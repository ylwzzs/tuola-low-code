#### 一、**功能背景**

员工间沟通时经常存在以下情况，如普通员工向高管频繁发送无关消息可能干扰决策效率，跨部门敏感信息泄露风险增加，或特定岗位（如财务、法务）需避免非必要沟通。传统通讯工具缺乏细粒度权限控制，导致企业面临信息泄露风险与沟通效率低下的双重挑战。

#### 二、**功能价值**

1. 精准权限控制：支持多维度配置，降低信息干扰。

2. 系统与个人规则结合：管理员全局管控与员工自定义规则互补，满足不同管理需求。

3. 合规与风险规避：限制敏感操作，防止信息泄露，满足审计与监管要求。

4. 避免无效沟通：通过场景化权限（如临时单聊），平衡紧急协作与长期权限管控，减少无效沟通。

#### 三、**功能概述**

1. 聊天权限限制范围包括：单聊 / 拉保护对象进群 / 群内@保护对象 / 必达保护对象 / 与保护对象音视频通话；

2. 聊天权限规则分为：系统级预设规则和前台个人级自定义规则，个人级别高于系统级别；

3. 个人级别：保护对象可在前台限制对象的个人详情页查看聊天限制情况；可在偏好设置中查看系统级为其预设的规则及配置个人级自定义规则（不允许单聊/不允许拉进群/不允许群内@/不允许必达/不允许音视频通话）；

4. 系统级预设规则：管理员可在后台预设规则，规则中包含保护对象、限制对象及限制哪些维度（不允许单聊/不允许拉进群/不允许群内@/不允许必达/不允许音视频通话）。

#### 五、**功能详细说明**

###### 1. 前台使用-个人沟通时限制

①单聊沟通限制时：“保护方”给“限制方”发送消息时，会提示：**对方没有权限无法回复消息**，并提供“解除限制”入口，解除后就可以正常聊天；

“限制方”给“保护方”发送消息时提示：**无法发送并提示无权限**，若此时有单聊申请权限则会显示发送申请，“限制方”发送申请时会提示申请的是长期单聊/临时单聊，申请发送后保护方/审批人收到系统消息提醒，此时可选择同意/拒绝，同意后便可进行聊天。

![](c0a96c1c8468890dd20933e27ed4a215.jpg)![](ff043804f4bc6a8d5a640adb4ae79c8a.jpg)

②拉人进群限制时：不允许拉人进群且不支持同意后进群时，“限制方”拉“保护方”进群则会提示：**邀请失败**；

若支持同意后入群时，“限制方”拉“保护方”进群则会提示已发送申请，“保护方”收到邀请进群的系统消息，可点击同意/拒绝（注意：群聊消息转话题、动态规则入群的不做限制）。

![](18327dcd3588fe5b174073a38a54a48f.jpg)

③群内@限制时：“限制方”在群内@保护方个人/@包含“保护方”的群身份时则无法@成功，消息变为普通消息体，“保护方”也不会收到@高亮提醒，且有灰色消息提示：你无权限@XXX（@全体成员不做限制）。

![](c5c438d61d30791412aee5aee863514a.jpg)

④必达限制时：不允许“限制方”给“保护方”发送必达，创建必达时，若接收人包含“保护方”则提示不允许，包括消息转必达、聊天工具栏发送必达、群消息未读详情转必达、会话顶部发送必达等场景。

![](d108b7a3fc45a76e3696ed0f436f7c3e.jpg)

⑤音视频通话限制时：不允许“限制方”给“保护方”发送音视频通话，选择“保护方”时提示无权限等，包括单聊/群聊聊天工具栏发起、会话顶部发起音视频通话等场景。

###### 2. 前台使用-个人详情页-聊天权限查看/配置

①保护对象可在限制对象的个人详情页查看对限制对象的限制情况，有个人自定义权限时可点击下方的权限设置进行配置。比如高管可以在此处查看对某个普通员工的聊天限制情况。

![](54dab863d794b0cc596b21ed08ab1a19.jpg)

②权限调整时，对于管理员在后台预设的规则，只能往严格方向调整，如管理员设置的不允许单聊但允许申请单聊时，此时用户可以调整为不允许申请单聊；

若管理员设置的不允许申请单聊，此时用户不能调整为允许申请单聊。若管理员未在后台设置有关规则，则用户可随意调整。

![](5db8322d445af46380e72e34c0886255.jpg) ![](9af98c01341ac2e339e08efd7752d37b.jpg)

###### 3. 用户设置-个人聊天权限查看/设置

①分为系统级及权限列表，系统级中显示后台预设规则中保护对象包含我的规则数据，只可查看不能编辑；

权限列表仅当后台配置中添加自定义权限用户包含自己时才显示该tab，可新建/编辑个人自定义规则，也可通过此列表查看系统级和个人级聚合限制数据。

当系统级规则及个人级规则冲突时，按照二者最严格的规则生效，可在限制对象的个人详情页查看实际生效规则。

比如当管理员预设的规则不够严格或者缺少时，用户可自己去新建规则。

![](2a0c12843fedc0f1012ff8feffe86661.jpg)

###### 4. 后台设置-聊天权限设置-基础设置

![](2f5dbe5df499726960cc5968553e1ec8.jpg)

①启用聊天权限控制：默认关闭，开启后，pc端前台偏好设置—显示聊天限制配置入口，app端设置—消息设置—显示聊天限制配置入口，按照配置的权限规则生效，关闭表示不限制聊天，前台权限入口不显示。

![](4b8bdc21db5fefcb53c7d7f256ee625a.jpg)

②单聊权限的审批人：默认为保护对象，也可设置指定人员进行审批；负责审批是否通过限制对象申请的单聊权限，批准后可按照配置的规则范围聊天；

当规则设置了允许申请单聊时，限制对象可通过点击会话中灰色消息提示“发送申请”，根据后台设置的审批人，审批人会收到系统消息提醒并可选择同意或拒绝。

比如有的公司是需要保护对象进行审批即可，而有的是需要经过人事审批才可进行沟通。

![](0e5e745468394b57b804cb114e5f1369.jpg)

③允许添加自定义权限的用户：聊天权限控制包括系统级权限及个人级权限；系统级权限是后台管理员预设的规则，普通用户无法修改；

个人级权限是用户在前台自定义的规则，可限制哪些人与自己聊天。管理员可后台配置哪些人有权限自定义个人规则，无权限则只能查看，不能新建个人规则。

比如有的公司是允许全体员工都可以配置限制谁与自己聊天，而有的公司则只允许一些高管才有权限去限制谁与自己聊天。

![](3561bc3b0b8184725b68c3f2181ec1fe.jpg)

###### 5. 后台设置-聊天权限设置-系统级配置

![](4510e6aaad14244fabe278126f53b900.jpg)

①系统预设规则：规则中包含保护对象、限制对象及限制哪些维度（不允许单聊/不允许拉进群/不允许群内@/不允许必达/不允许音视频通话）。

比如可以提前为一些高管预设好规则，就不用麻烦高管自己去一个个配置了。

![](f971583d355ac2ebbc54ca59d164b4ca.jpg)

l 保护对象：保护对象指不被别人打扰，如高层领导。可设置包含/排除人员、部门、分部、角色、岗位、所有人等，也可在详情中根据人员安全级别进行配置。

![](d7514ea2318846dd3e5c57cc9308050b.jpg)

l 限制对象：限制对象指限制不能打扰别人，如普通员工。可设置包含/排除人员、部门、分部、角色、岗位、所有人等，也可在详情中根据人员安全级别进行配置。

l 不允许单聊：勾选后显示允许申请单聊、允许单聊回复开关；

允许申请单聊开关，默认关闭，开启后可设置X天内只能申请一次，最少1天，申请单聊的类型包括长期单聊和临时单聊，临时单聊可设置申请通过X小时后恢复聊天限制。

![](b3535e6d9f636b12917d92d582b17182.jpg)

l 不允许拉进群：勾选后显示仅支持同意后进群（默认未选中）；不允许被拉进群，被限制方发起群聊包含保护方时，提示XX人员邀请失败；

不允许被拉进群同时选中仅支持同意后入群，则被限制方发起群聊包含保护方时，提示邀请XX人员已发送申请，对应人员收到系统消息，可点击同意/拒绝（群聊消息转话题、动态规则入群的不做限制）。

![](29bf29849a2043eedb7b6d9d3ccc5645.jpg)

l 不允许群内@：勾选后不允许限制对象@保护对象，包括：

@保护方个人、包含"保护方"的群身份；

限制方@“保护方”，发送至会话的消息为普通消息体，保护方也不会收到@高亮提醒，且有灰色消息提示：你无权限@XXX（@全体成员不做限制）。

![](11cc479d37c67583de1f98d70a635a20.jpg)

l 不允许必达：勾选后不允许“限制对象”给“保护对象”发送必达；包括消息转必达、聊天工具栏发送必达、群消息未读详情转必达、会话顶部发送必达等场景。

![](97fa78c07ba66bd566b03bdb20811c44.jpg)

l 不允许音视频通话：勾选后不允许"限制对象"给"保护方"发送音视频通话，包括单聊/群聊聊天工具栏发起、会话顶部发起音视频通话等场景。

②权限查询：搜索中选择"保护对象"及"限制对象"，即可展示二者间生效规则，包括最终生效规则情况、用户个人级规则情况及系统预设规则情况。

![](e36959c089da91e0050ae184a27c38a3.jpg)

③系统级查询：搜索中选择"保护对象"/"限制对象"，可查询系统预设规则中包含"保护对象"/"限制对象"的相关数据。

![](160002daa2e517c5e3e48a6face097e5.jpg)

④重置申请状态：会将该组织下所有已经申请单聊并通过的权限进行重置清除，恢复为未申请的情况。

比如普通员工申请与高管沟通，高管同意申请后，后面又想限制时，就可以重置申请状态，此时需要再次申请并通过才可继续沟通。

![](ffde124d58ac8730af48399674ef9a12.jpg)

#### 六、**更多场景示例**

###### 1. 高管保护

普通员工向董事长/总经理等高管汇报时，没有单聊权限，需提交单聊申请，由指定人审批并通过后才可进行沟通汇报。

![](54e436fd5610b0baee07b80ac3920bb0.jpg) ![](3101ed220cf25098c1a8ed036bef5742.jpg)

###### 2. 跨部门协作

销售部员工可申请与技术部员工临时沟通项目进度，但禁止长期私聊、必达及进行音视频通话。

![](0fd1a7fbd0e17e83a107d7cd0bd39860.jpg)![](e24fb9ecf9a7fd4c4c32c3cd097a0d6a.jpg)

###### 3. 群聊管控

禁止市场团队拉技术同事进群并在群内@技术同事，避免干扰技术同事日常工作，影响效率。

![](09d97c1da06dd7d17e11a41237753ea9.jpg)![](908ce72458f158d1ed45a0bba407999f.jpg)

###### 4. 敏感岗位隔离

法务人员仅允许与同部门同岗位进行日常沟通交流，与其他人员皆不可进行沟通。

![](49ec15dcf5a6630afa116b97a7263f25.jpg)

#### 八、**注意点**

1. 当系统级规则与个人级规则冲突时，按照“最严格原则”生效（例如系统级禁止私聊 + 个人级允许临时聊天 → 最终禁止私聊）；

2. 开启允许申请单聊时，限制申请次数不要太低，防止恶意刷申请，对保护方造成困扰；

3. 同一保护对象及限制对象尽量不要创建多个规则，会导致性能问题且不太易于理解；

4. 目前限制规则仅对内部成员生效，外部用户暂不生效。
